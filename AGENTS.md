# Repository `AGENTS.md`

This repository contains the `julia-actions/setup-julia` GitHub Action.

## Scope

- Prefer small, reviewable changes.
- Preserve the action contract in [`action.yml`](/Users/ai/work/setup-julia/action.yml):
  - runtime is `node24`
  - checked-in action entrypoint is `dist/index.js`

## Node and Package Manager

- The Node version is pinned in [`.tool-versions`](/Users/ai/work/setup-julia/.tool-versions).
- When running Node or npm locally, use that exact version.
- If `mise` is available, prefer:
  - `mise exec nodejs@$(awk '{print $2}' .tool-versions) -- npm ...`
- Do not assume the host `node` or `npm` version is correct.

## Build Outputs

- `src/` contains the TypeScript source.
- `lib/` is generated by `npm run build` (`tsc`) and is checked in.
- `dist/index.js` is the bundled GitHub Action artifact produced by `npm run pack` (`tsup`) and is checked in.
- Keep `dist/` to a single output file: `dist/index.js`.
- The checked-in `dist/index.js` must remain CommonJS.
- If you change runtime code or bundling configuration, regenerate both `lib/` and `dist/`.

## Common Commands

- Install dependencies: `mise exec nodejs@<version> -- npm ci`
- Build TypeScript: `mise exec nodejs@<version> -- npm run build`
- Run tests: `mise exec nodejs@<version> -- npm test`
- Build checked-in bundle: `mise exec nodejs@<version> -- npm run pack`
- Full local verification for most changes:
  - `npm run build`
  - `npm test`
  - `npm run pack`

## CI Expectations

- CI checks that generated files are up to date.
- CI also checks that `npm install` does not modify tracked files unexpectedly.
- Before finishing a change that touches Node dependencies, source, or bundling:
  - run `npm run build`
  - run `npm test`
  - run `npm run pack`
  - inspect `git diff`

## Dependency Changes

- Keep direct runtime dependencies in `dependencies` only if they are used by the action at runtime.
- Test-only packages belong in `devDependencies`.
- Because the action runtime is CommonJS but some modern dependencies are ESM-only, verify that the final `dist/index.js` does not rely on runtime `require()` calls for ESM-only packages.

## Editing Guidance

- Prefer changing files in `src/` rather than editing generated files by hand.
- Do not hand-edit `lib/` or `dist/` unless there is a strong reason; regenerate them instead.
- Preserve existing behavior across:
  - version resolution
  - architecture handling
  - cache usage
  - nightly and prerelease handling

## Commit Splitting

- Any changes to `package-lock.json` should go in their own commit.
- A commit that changes `package-lock.json` should not include unrelated files.
- The commit message for a `package-lock.json`-only commit should start with `` `package-lock.json` only: ``.
- Any changes to `lib/*` and `dist/*` should go in their own commit.
- A commit that changes `lib/*` and `dist/*` should not include unrelated files.
- The commit message for a generated-artifacts-only commit should start with `` `lib/*` and `dist/*` only: ``.

## Pull Requests

- AI tools should add themselves as co-authors on all commits they make.
- Whenever an AI tool writes a PR body or opens a PR, the PR body must mention the AI tool as a co-author of the PR.
- Include a brief summary of what changed and why.
- If the scope changes after review feedback, update the PR title and body to match.
